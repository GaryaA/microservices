# Summary: Section 5 - Deep Dive on Cloud Native Apps & 15-Factor Methodology

## Обзор секции

В пятой секции курса изучались принципы Cloud Native приложений и методология 15-Factor App. Были рассмотрены best practices для создания приложений, готовых к работе в облачных окружениях.

## Что было изучено

### 1. Cloud Native концепция
- Приложения, разработанные специально для облака
- Использование преимуществ облачных платформ
- Горизонтальное масштабирование
- Отказоустойчивость и самовосстановление

### 2. 12-Factor App (расширено до 15-Factor)
- Оригинальная методология от Heroku
- Расширение до 15 факторов для современных приложений
- Принципы для создания масштабируемых приложений

### 3. 15-Factor Methodology

#### I. Codebase
- Один код для всех окружений
- Версионирование через Git
- Разные конфигурации для разных окружений

#### II. Dependencies
- Явное объявление зависимостей
- Не полагаться на системные библиотеки
- Использование dependency managers (Maven, Gradle)

#### III. Config
- Хранение конфигураций в переменных окружения
- Не хранить конфигурации в коде
- Разные конфигурации для разных окружений

#### IV. Backing Services
- Внешние сервисы как attached resources
- Легкая замена сервисов
- Без изменений в коде

#### V. Build, Release, Run
- Разделение стадий: build, release, run
- Build создает артефакт
- Release добавляет конфигурацию
- Run запускает приложение

#### VI. Processes
- Stateless процессы
- Не хранить состояние в памяти
- Горизонтальное масштабирование

#### VII. Port Binding
- Приложение экспортирует сервисы через порты
- Не полагаться на веб-сервер для инъекции портов
- Self-contained приложения

#### VIII. Concurrency
- Масштабирование через процессы
- Не через threads
- Использование process model

#### IX. Disposability
- Быстрый старт и graceful shutdown
- Минимизация startup time
- Graceful shutdown для сохранения данных

#### X. Dev/Prod Parity
- Минимизация различий между окружениями
- Использование одинаковых технологий
- Избежание "works on my machine"

#### XI. Logs
- Логи как потоки событий
- Не управление файлами логов
- Централизованная агрегация логов

#### XII. Admin Processes
- Административные задачи как одноразовые процессы
- Запуск в том же окружении
- Использование тех же зависимостей

#### XIII. API First
- API как первый класс
- Версионирование API
- Документирование API

#### XIV. Telemetry
- Мониторинг и observability
- Метрики, логи, трассировка
- Health checks

#### XV. Authentication and Authorization
- Безопасность с первого дня
- OAuth2, JWT
- Централизованное управление доступом

## Что было реализовано

### Применение принципов
- Использование переменных окружения для конфигураций
- Stateless микросервисы
- Health checks через Actuator
- Централизованное логирование
- API-first подход
- Безопасность через OAuth2/JWT

### Cloud Native практики
- Контейнеризация для портативности
- Использование внешних сервисов (БД, message brokers)
- Горизонтальное масштабирование
- Graceful shutdown

## Технические детали

### Spring Boot Cloud Native Features
- Actuator для health checks
- Externalized configuration
- Profile-based configuration
- Graceful shutdown support

### Container Best Practices
- Multi-stage builds
- Minimal base images
- Non-root users
- Health checks

## Best Practices

1. **Configuration Externalization** - конфигурации вне кода
2. **Stateless Design** - не хранить состояние
3. **Health Checks** - мониторинг состояния
4. **Logging** - структурированное логирование
5. **Security** - безопасность с первого дня
6. **Observability** - метрики, логи, трассировка
